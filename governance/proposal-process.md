# Proposal Process

## Overview
Any contributor may propose a change to laws, concepts, specification text, schemas, or examples.

## Required proposal content
- title and summary
- problem statement
- proposed change
- classification (clarifying, normative, breaking)
- compatibility impact
- affected files

## Process
1. Draft proposal and open a review request.
2. Discuss scope and classify change type.
3. Revise with reviewer feedback.
4. Record decision using a decision record.
5. Merge with changelog and version updates as needed.
