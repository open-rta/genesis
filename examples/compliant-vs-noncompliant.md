# Compliant vs Noncompliant Runtime Patterns

## Compliant pattern (Open RTA aligned)
A runtime exposes discoverable evidence artifacts, including:
- declared objective and authority artifacts
- trace events tied to objective and actions
- oversight records for operational visibility
- control records proving intervention capability
- replay/reconstruction records
- a manifest and/or conformance report listing evidence references

This runtime can substantiate attribution, observability, interruptibility, and replayability.

## Noncompliant pattern
A runtime "runs agents" but provides:
- no declared objective artifact
- no authority lineage
- no control intervention evidence
- no replay/reconstruction record
- no manifest or conformance report tying evidence together

This runtime cannot demonstrate Open RTA alignment even if it performs useful tasks.
