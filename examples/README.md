# Examples

This directory provides minimal Open RTA evidence artifacts and a manifest-first workflow.

## Core artifact examples

- `minimal-objective.json`
- `minimal-authority.json`
- `minimal-trace.json`
- `minimal-replay-record.json`
- `minimal-oversight-record.json`
- `minimal-control-record.json`
- `minimal-conformance-report.json`

Legacy YAML variants are included for readability:
- `minimal-objective.yaml`
- `minimal-authority.yaml`

## Manifest + report examples

- `open-rta-manifest.json`: sample manifest referencing evidence artifacts.
- `validation-report.json`: sample machine-readable output from `validate`.
- `generated-manifest.json`: starter manifest generated by `init-manifest` in non-interactive mode.
- `generated-validation-report.json`: validation output for `generated-manifest.json`.

## Example workflow

From `tools/validator`:

```bash
# generate starter manifest interactively
go run . init-manifest

# validate a manifest
go run . validate --manifest ../../examples/open-rta-manifest.json --report ../../examples/validation-report.generated.json
```

## Notes

These examples are illustrative only.

Real runtimes may store evidence in any internal layout. Open RTA requires discoverable evidence, not prescribed internal folder names.
